# chaos-sim 計画書

## 概要

ChaosTCG 一人回しシミュレーター。ws-simと同等の機能をChaos向けに実装する。

## データ方針

- **正（マスタデータ）**: 遊々亭（yuyu-tei.jp）のカード情報
- **インデックスキー**: カード番号（例: NP-003, YZ-401）を全データの主キーとして使用
- ローカル画像、レシピ、デッキ構築すべてカード番号で紐付ける
- 遊々亭の商品IDはURL参照用であり、内部的にはカード番号で管理する

## テーマ

- カラー: **青と白**（ChaosTCGのブランドカラー）
- 背景: ダークネイビー（#1a2332）
- アクセント: ブルー（#4a9fd4）
- ボーダー/テキスト: 白系（#e8f0ff）
- ステージ枠: 青（#3a7bd5）

## パートナーキャンセル（ダメージキャンセル）

WSのCXキャンセルに相当する仕組み。

1. プレイヤーにダメージ → デッキ上から1枚ずつ控え室へ（ダメージ1点につき1枚）
2. めくったカードが**自分のパートナーと同じカード名**だった場合、オートレベルアップ可能（任意）
3. オートレベルアップ実行時:
   - パートナーが表スタンドに戻る
   - 累積ダメージ0
   - そのカードがレベルカードとしてパートナーに重なる
   - **残りダメージが全て0になる（キャンセル）**

キャンセル確率 = デッキ残り中のパートナー同名カード枚数 / デッキ残り枚数

## デッキ構築画面

デッキログが存在しないため、自前のデッキ構築UIが必要。

### 機能
- カード番号 or カード名で検索（card_detailsテーブル参照）
- 枚数指定して追加
- パートナー指定（1種）
- ローカルストレージに保存/読み込み
- JSON形式でエクスポート/インポート

### デッキレシピスクリーンショットからのインポート

ローカルの `shots/recipe/` に323枚のデッキレシピスクリーンショットがある。
公式サイトのテーブル形式（番号/カード名/枚数）で、OCRで高精度に読み取り可能。

**検証結果:**
- カード番号列: Tesseract(eng)で95%以上の精度
- 枚数列: ほぼ100%
- テーブル位置: 全レシピで共通レイアウト（x=270-430, y=830〜）

**実装方針:**
1. 全323枚のレシピ画像を事前にOCR処理
2. カード番号+枚数のペアを抽出 → JSONファイルとして保存
3. chaos-simのデッキ構築画面から既成デッキとして選択可能に
4. カード番号はcard_urlsテーブルと突合してカード名を補完

## デッキレシピデータベース（recipe.html）

デッキ構築画面とは別に、レシピ閲覧・検索・インポート用の独立ページを用意する。

### データ構造（recipes.json）

```json
[
  {
    "id": "2018_akita_1",
    "source_file": "-recipe-2018_akita.png",
    "tournament": "ChaosTCG 日本選手権2018 ネオスタンダード in 秋田",
    "year": 2018,
    "event_type": "日本選手権",
    "location": "秋田",
    "player": "ミミィ…@ぶるたぶFC",
    "rank": 1,
    "partner": "YZ-401",
    "partner_name": "才色兼備な学生会長「三司 あやせ」",
    "os": "ゆずソフト",
    "format": "ネオスタンダード",
    "cards": [
      {"number": "YZ-401", "count": 4},
      {"number": "YZ-408", "count": 2},
      ...
    ],
    "total": 20
  }
]
```

### ページ機能（recipe.html）

- 大会名/年/地域/OS/パートナーで絞り込み検索
- レシピ一覧表示（大会、順位、プレイヤー、パートナー）
- レシピ詳細表示（カードリスト、元画像リンク）
- 「このデッキをインポート」ボタン → ローカルストレージに保存 → sim画面で使用可能

### OCRバッチ処理

1. メタ情報抽出（大会名、HN、順位、パートナー）: y=250-800
2. カードリスト抽出（番号+枚数）: y=800〜末尾
3. 1ページに複数デッキが含まれる場合あり（1位〜3位等）
4. 結果を recipes.json に出力
5. 手動で精度確認・修正

## WSとの主な差異

| 項目 | WS | Chaos |
|------|-----|-------|
| 舞台構成 | 前列3+後列3=6枠 | 前衛2+後衛2+パートナー1=5枠 |
| デッキ枚数 | 50枚 | 20枚（メイン）+ エクストラ |
| ダメージ | ソウル→クロック→レベル | バトル（ATK vs BP）→裏 |
| 勝利条件 | Lv4到達で敗北 | パートナーのBP0で敗北 |
| CX/トリガー | CXでキャンセル | なし |
| 特殊領域 | 思い出、クロック、レベル、ストック | バックヤード |
| カード状態 | スタンド/レスト/リバース | 表スタンド/表レスト/裏 |
