# chaos-sim 計画書

## 概要

ChaosTCG 一人回しシミュレーター。ws-simと同等の機能をChaos向けに実装する。

## データ方針

- **正（マスタデータ）**: 遊々亭（yuyu-tei.jp）のカード情報
- **インデックスキー**: カード番号（例: NP-003, YZ-401）を全データの主キーとして使用
- ローカル画像、レシピ、デッキ構築すべてカード番号で紐付ける
- 遊々亭の商品IDはURL参照用であり、内部的にはカード番号で管理する

## テーマ

- カラー: **青と白**（ChaosTCGのブランドカラー）
- 背景: ダークネイビー（#1a2332）
- アクセント: ブルー（#4a9fd4）
- ボーダー/テキスト: 白系（#e8f0ff）
- ステージ枠: 青（#3a7bd5）

## パートナーキャンセル（ダメージキャンセル）

WSのCXキャンセルに相当する仕組み。

1. プレイヤーにダメージ → デッキ上から1枚ずつ控え室へ（ダメージ1点につき1枚）
2. めくったカードが**自分のパートナーと同じカード名**だった場合、オートレベルアップ可能（任意）
3. オートレベルアップ実行時:
   - パートナーが表スタンドに戻る
   - 累積ダメージ0
   - そのカードがレベルカードとしてパートナーに重なる
   - **残りダメージが全て0になる（キャンセル）**

キャンセル確率 = デッキ残り中のパートナー同名カード枚数 / デッキ残り枚数

## デッキ構築画面

デッキログが存在しないため、自前のデッキ構築UIが必要。

### 機能
- カード番号 or カード名で検索（card_detailsテーブル参照）
- 枚数指定して追加
- パートナー指定（1種）
- ローカルストレージに保存/読み込み
- JSON形式でエクスポート/インポート

### デッキレシピスクリーンショットからのインポート

ローカルの `shots/recipe/` に323枚のデッキレシピスクリーンショットがある。
公式サイトのテーブル形式（番号/カード名/枚数）で、OCRで高精度に読み取り可能。

**検証結果:**
- カード番号列: Tesseract(eng)で95%以上の精度
- 枚数列: ほぼ100%
- テーブル位置: 全レシピで共通レイアウト（x=270-430, y=830〜）

**実装方針:**
1. 全323枚のレシピ画像を事前にOCR処理
2. カード番号+枚数のペアを抽出 → JSONファイルとして保存
3. chaos-simのデッキ構築画面から既成デッキとして選択可能に
4. カード番号はcard_urlsテーブルと突合してカード名を補完

- 「このデッキをインポート」ボタン → ローカルストレージに保存 → sim画面で使用可能

### OCRバッチ処理（v2: 罫線検出方式）

現行のv1（全幅OCR+ヘッダー区切り）は精度に限界がある。v2では以下の手法に切り替える。

#### 手法概要

1. **罫線検出**: テーブル範囲(x=280-960)内で水平方向に暗いピクセルが80%以上の行を罫線として検出
2. **行分割**: 罫線間をデータ行として切り出し（行高さ25-45px）
3. **カテゴリ判定**: 行の背景色の彩度で分類
   - 彩度>80 → カテゴリヘッダー（◆キャラ/イベント/セット/エクストラ◆）
   - 彩度<80 → データ行
4. **デッキ境界検出**: 白い大きな行(h>55, brightness>240)が新デッキの開始
5. **セル分割OCR**: 番号セル(左端85px)と枚数セル(右端55px)を別々にOCR（psm 7, グレースケール）
6. **後処理**: O→0, S→5, I→1, L→1 の補正

#### カテゴリ行の背景色

| カテゴリ | 背景色 | 判定条件 |
|----------|--------|----------|
| キャラカード | ピンク (255,200,200) | R高, sat>80 |
| イベントカード | 緑 (134,236,122) | G>R, G>B |
| セットカード | 紫 (176,141,176) | B>G, min<160 |
| エクストラカード | オレンジ (251,151,100) | R>G, B<150 |

#### 検証結果（2026-05-17）

| 画像 | デッキ数 | 正常(M50+E10) | 精度 |
|------|----------|---------------|------|
| 2018_akita | 3 | 2/3 | 67% |
| 2018_hamamatsu | 8 | 5/8 | 63% |
| 2018_tokyo_a | 9 | 6/9 | 67% |
| 2018_kagoshima | 4 | 3/4 | 75% |
| 2019_kyoto | 4 | 3/4 | 75% |
| 2019_makuhari | 6 | 6/6 | 100% |
| bcf2010_sz | 1 | 1/1 | 100% |
| **合計** | **35** | **26/35** | **74%** |

※異常デッキの主因: デッキ境界の誤検出（白い行が途中にある）、パートナー行の欠落

#### メタ情報抽出

デッキ境界の白い行ブロック内に以下の情報がある:

- **使用OS（作品名）**: 左1/3セル → 精度80-90%
- **HN（プレイヤー名）**: 中央セル → 精度70-80%（短い英字名は誤読しやすい）
- **順位**: 右セル → 精度95%+（「N位」パターン）
- **参加大会**: 別行 → 精度90%+

メタ情報テーブルの構造は2パターン:
1. **個別デッキページ型**（bcf2010等）: 「デッキ名 | HN | 順位」が1行
2. **一覧ページ型**（2018以降）: 各デッキの前に「使用OS | HN | 順位」行

## フィールドカード構造（スロット設計）

### データ構造

各エリア（5枠: パートナー, 前衛L, 前衛R, 後衛L, 後衛R）は以下の構造を持つ:

```js
slot = {
  chara: Card,          // 一番上のキャラ（表示・戦闘に使う）
  levels: [Card, ...],  // レベルカード（枚数 = レベル）
  sets: [Card, ...],    // セットカード（複数可）
}
```

- `chara`: 現在そのエリアを占めるキャラ。表示・攻撃力/耐久力の基本値はこのカードを参照
- `levels`: レベルカードの配列。`levels.length` = そのキャラのレベル
- `sets`: セットカードの配列。複数枚セット可能

### レベルアップの種類

| 種類 | 対象 | 操作 | 結果 |
|------|------|------|------|
| 通常レベルアップ | パートナー | 手札から同名カードを重ねる | 使ったカード → levels に追加 |
| エクストラキャラ登場 | フレンド/パートナー | EXデッキからエクストラキャラを重ねる | 元のchara → levels に移動、エクストラキャラが新chara |
| エクストラキャラのレベルアップ | エクストラキャラ | EXデッキから同名エクストラを重ねる | 使ったカード → levels に追加 |
| オートレベルアップ | パートナー | ダメージ処理中に同名カードが出る | そのカード → levels に追加、残ダメージ0 |

全パターン共通: **重ねたカードがレベルカードになり、レベル数+1**

### 攻撃力/耐久力の計算

```
最終攻撃力 = chara.atk + Σ(各レベルカードの atk_mod)
最終耐久力 = chara.bp  + Σ(各レベルカードの bp_mod)
```

- 補正値はレベルカード1枚ごとに、**そのレベルカード自身の補正値**を加算
- パートナーカード/エクストラキャラは補正値を持つ（card_details.atk_mod, bp_mod）
- 通常キャラカードは補正値を持たない（`-`）

### セットカード

- キャラにセットする（複数可）
- セットカードはキャラと一緒に移動する
- キャラが控え室に行く場合、セットカードも控え室に行く

### レンダリング

- スロット表示: chara の画像を表示
- レベルバッジ: `Lv{levels.length}` を右上に表示（levels.length > 0 の場合）
- セットバッジ: `S{sets.length}` を左下に表示（sets.length > 0 の場合）
- 右クリックメニュー:
  - 「セットを見る」→ セット一覧パネル（DD で取り外し可能）
  - 「レベルを見る」→ レベルカード一覧

### DD操作

- セットカード（type=set）をスロットにドロップ → `sets` に追加
- エクストラキャラをスロットにドロップ → 元chara を `levels` に移動、ドロップしたカードが新 `chara`
- 通常キャラをスロットにドロップ → 既存キャラ+レベル+セットを控え室に送り、新キャラとして配置
- パートナースロットに同名カードをドロップ → レベルアップ（`levels` に追加）

## WSとの主な差異

| 項目 | WS | Chaos |
|------|-----|-------|
| 舞台構成 | 前列3+後列3=6枠 | 前衛2+後衛2+パートナー1=5枠 |
| デッキ枚数 | 50枚 | 20枚（メイン）+ エクストラ |
| ダメージ | ソウル→クロック→レベル | バトル（ATK vs BP）→裏 |
| 勝利条件 | Lv4到達で敗北 | パートナーのBP0で敗北 |
| CX/トリガー | CXでキャンセル | なし |
| 特殊領域 | 思い出、クロック、レベル、ストック | バックヤード |
| カード状態 | スタンド/レスト/リバース | 表スタンド/表レスト/裏 |
