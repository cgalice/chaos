# カードデータ取得設計

## データソース

遊々亭（yuyu-tei.jp）のChaosTCG販売ページからスクレイピングで取得する。

## URL構造

```
カード個別: https://yuyu-tei.jp/sell/chaos/card/{弾コード}/{商品ID}
弾一覧:    https://yuyu-tei.jp/sell/chaos/s/{弾コード}
```

### 弾コード例

| 弾コード | 弾名 |
|---------|------|
| ntp1.0 | ニトロプラス1.00 |
| ntp2.0 | ニトロプラス2.00 |
| acp1.0 | アクアプラス1.00 |
| sw1.0 | ストライクウィッチーズ1.00 |
| sc1.0 | STEINS;GATE & CHAOS;HEAD |

## 設計方針

- 必要になったタイミングでオンデマンドに取得する
- 取得したデータはローカルにキャッシュ（JSONL形式）
- 全件一括取得は行わない（サーバー負荷考慮）
- キャッシュファイル: `develop/cards_cache.jsonl`

## カード種類の判別

### 遊々亭の検索フィルター

カード種類は以下のURLパラメータで絞り込み可能:

```
https://yuyu-tei.jp/sell/chaos/s/search?search_word=&rare=&type={種類}&kizu=0
```

| パラメータ値 | カード種類 | 説明 |
|-------------|-----------|------|
| Chara | キャラカード | メインデッキに入る |
| Event | イベントカード | メインデッキに入る |
| Set | セットカード | メインデッキに入る |
| Extra | エクストラカード | エクストラデッキに入る |
| Partner | パートナーカード | メインデッキに1枚 |

**注意事項:**
- パラメータは `type=Extra`（単数形）。`types[]=Extra`（複数形・配列）ではフィルターが効かない
- `kizu=0` を付けないと在庫あり商品のみに制限される
- 1回のレスポンスで約500件が上限。全件取得するにはver_codeとの組み合わせが必要
- `sell_ok_only=0` を付けると在庫なし含む全件が返るが、typeフィルターが無効になる

### 判別ロジック（優先順）

1. **card_urls.card_type** — 遊々亭フィルターから取得した正解データ（最も信頼性が高い）
2. **effect_text** — `【エクストラ】`で始まる or `エクストラデッキのこのカード`を含む → Extra
3. **レアリティ** — `EXR`, `EX`, `EXSP` → Extra専用レアリティ
4. **攻撃力補正/耐久力補正** — 値がある → Partner（パートナーまたはエクストラキャラ）
