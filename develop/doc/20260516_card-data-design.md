# カードデータ取得設計

## データソース

遊々亭（yuyu-tei.jp）のChaosTCG販売ページからスクレイピングで取得する。

## URL構造

```
カード個別: https://yuyu-tei.jp/sell/chaos/card/{弾コード}/{商品ID}
弾一覧:    https://yuyu-tei.jp/sell/chaos/s/{弾コード}
```

### 弾コード例

| 弾コード | 弾名 |
|---------|------|
| ntp1.0 | ニトロプラス1.00 |
| ntp2.0 | ニトロプラス2.00 |
| acp1.0 | アクアプラス1.00 |
| sw1.0 | ストライクウィッチーズ1.00 |
| sc1.0 | STEINS;GATE & CHAOS;HEAD |

## 設計方針

- 必要になったタイミングでオンデマンドに取得する
- 取得したデータはローカルにキャッシュ（JSONL形式）
- 全件一括取得は行わない（サーバー負荷考慮）
- キャッシュファイル: `develop/cards_cache.jsonl`

## カード種類の判別

### 遊々亭の検索フィルター

カード種類は以下のURLパラメータで絞り込み可能:

```
https://yuyu-tei.jp/sell/chaos/s/search?search_word=&rare=&type={種類}&kizu=0
```

| パラメータ値 | カード種類 | デッキ区分 | 説明 |
|-------------|-----------|-----------|------|
| Chara | キャラカード | メインデッキ | フィールドに登場するキャラ |
| Event | イベントカード | メインデッキ | 使い切り効果 |
| Set | セットカード | メインデッキ | キャラに装備 |
| Extra | エクストラカード | エクストラデッキ | エクストラキャラ＋スペシャルエクストラ |
| Partner | パートナーカード | メインデッキ(1枚) | パートナー専用キャラ |

#### エクストラカードの定義

エクストラデッキに入るカード。2種類ある:

1. **エクストラキャラ** — 攻撃力/耐久力を持ち、フィールドに登場できる。effect_textが`【エクストラ】〔条件〕`で始まる
2. **スペシャルエクストラ** — 攻撃力/耐久力を持たず、フィールドに登場しない。effect_textに`エクストラデッキのこのカード`を含む

#### ページネーション仕様

- `page=`パラメータなし → 1ページ目（約500件）
- `page=2`〜`page=N` → 2ページ目以降
- **`page=1`を明示すると0件が返る（バグ）** → 1ページ目はパラメータなしで取得すること
- 各ページの件数は一定ではない（400〜540件程度）
- 最終ページはページネーションリンクから判断（`page=N`のリンクがある）
- `kizu=0`必須（傷なし商品のみ=在庫制限を外す）

**注意事項:**
- `types[]=Extra`（複数形・配列）ではフィルターが効かない。`type=Extra`（単数形）が正しい
- `sell_ok_only=0` を付けるとtypeフィルターが無効になる（全件返る）

### 判別ロジック（優先順）

1. **card_urls.card_type** — 遊々亭フィルターから取得した正解データ（最も信頼性が高い）
2. **effect_text** — `【エクストラ】`で始まる or `エクストラデッキのこのカード`を含む → Extra
3. **レアリティ** — `EXR`, `EX`, `EXSP` → Extra専用レアリティ
4. **攻撃力補正/耐久力補正** — 値がある → Partner（パートナーまたはエクストラキャラ）
